# Flake check timings — AFTER PHASE 5 (coverage split)

Phase 5 patches:

- `packages.ncps` is now a lean binary build (`doCheck = false`, single
  output, no preCheck plumbing, no integration backends in
  `nativeBuildInputs`).
- New `packages.ncps-coverage` (separate file under
  `nix/packages/ncps-coverage/`) carries the test scaffold: all four
  backends in `nativeBuildInputs`, full preCheck/postCheck, multi-output
  `[ out coverage ]`, `doCheck = true` with `-coverprofile`.
- `packages.ncps.passthru.coverage = packages.ncps-coverage.coverage`
  so `nix build .#ncps.coverage -L` continues to resolve to a
  multi-output build that produces `result-coverage` — the contract the
  shared CI workflow at `kalbasit/gh-actions/.github/workflows/build.yml`
  relies on is preserved verbatim.
- `nix/checks/flake-module.nix` `mkCohort` re-adds the integration
  backends (`garage`, `mariadb`, `postgresql`, `redis`, `python3`,
  `awscli2`, etc.) as `nativeBuildInputs` so the per-backend cohorts
  can still spin their services up; `packages.ncps` no longer carries
  them.
- `ncps-coverage` is filtered out of the `checks` attrset so
  `nix flake check` does NOT pay for the ~12m coverage run. (Phase 6
  will generalize this into a full explicit enumeration.)
- `nix/packages/docker.nix` no longer overrides `doCheck` or
  `outputs` on its embedded ncps reference — the lean `packages.ncps`
  already satisfies both.

## Headline numbers — what changed in Phase 5

`nix build .#ncps`: **12m → 1m10s** (verified cold).
`nix flake check` no longer includes the ~12m monolith via
`packages.ncps`. The lean ncps build is still in checks via
`// self'.packages` (Phase 6 will replace that whole block with an
explicit enumeration), but at 1m10s it's no longer the bottleneck.
`packages.ncps-coverage` is filtered out of checks so coverage is
only paid by the explicit CI step `nix build .#ncps.coverage`, on
non-fork PRs.

## Wall-clock model

Pre-Phase-5 (`nix flake check`):

    max(ncps monolith 12m, cohort group ~5m, lint/drift ~1m, ...) ≈ 12m

Post-Phase-5 (this state):

    max(ncps lean 1m, cohort group ~5m, lint/drift ~1m, ...) ≈ 5m

That's the headline win this whole change was aiming at: **flake-check
wall-clock cut from ~12m to ~5m, in CI as measured by the shared
build job that runs `nix flake check`.** The remaining floor is the
`ncps-mysql-tests` cohort plus Docker image build (the latter is
unaffected by this work and is part of the same CI job by design at
`kalbasit/gh-actions/.github/workflows/ci.yml`).

## What's left

Phase 6 (`/nix/checks/flake-module.nix`): replace the
`self'.packages // self'.devShells //` merge with an explicit
enumeration. Removes the docker/k8s-tests/treefmt/etc. derivations
from the checks attrset entirely (they aren't quality gates), and
drops the targeted `ncps-coverage` filter introduced here.
