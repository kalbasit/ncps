## What `nix flake check` no longer pays for

Eight derivations that used to be in the checks attrset via
`self'.packages // self'.devShells //` are no longer evaluated by
`nix flake check`. None of them were quality gates; they were just
packages that happened to be in `self'.packages`:

- `default` ŌĆö alias of `ncps`
- `deps` ŌĆö `nix run .#deps` process-compose dev dependencies
- `docker`, `docker-dev` ŌĆö runtime images, built explicitly by CI
- `push-docker-image` ŌĆö CLI for the CI image push step
- `k8s-tests` ŌĆö CLI for local Kind testing
- `update-cu-base` ŌĆö CLI for refreshing the container-use base image
- `ncps-coverage` ŌĆö coverage build, invoked only by CI's codecov step

All eight remain buildable directly as `nix build .#<name>`.
